# スコアキャプション編集ツール（PWA）

iPhoneで撮影した試合動画に、得点・チーム名・セット・サーブ権などのスコアボードを重ねて表示するための **編集データ（JSON）** を作成する PWA です。動画を見ながら得点タイミングを記録し、動画上にスコアボードを重ねたプレビューを確認できます。

> 最終的な MP4 書き出しは行いません。書き出した JSON を使って、PC 側（FFmpeg 等）で動画に焼き込む想定です（次フェーズ）。

## 特長
- **ビルド不要**：素の HTML/CSS/JS（ES Modules）。静的ファイルを置くだけ。
- **オフライン動作**：Service Worker でアプリ本体をキャッシュ。
- **端末外にアップロードしない**：動画はブラウザ内で再生するだけ。サーバ送信なし。
- **動画は保存しない（再選択方式）**：編集データのみ IndexedDB に保存。再開時に同じ動画を選び直します（iOS Safari の容量・自動削除対策）。

## 主な機能
- 得点入力（自/相手 +1・取り消し・前/次セット・サーブ権切替）と動画プレビュー追従
- **精密シーク**（◀10秒/◀1秒/▶1秒/▶10秒）＋**再生速度**（1x/0.5x/0.25x）
- **前の得点/次の得点へジャンプ**、**シークバー上の得点マーカー**（タップで移動）
- **得点履歴の編集**（時刻±0.5秒・加点チーム入替・削除／積算スコアは自動再計算）
- スコアボードは**全セットを横並びの列**で表示。**セットカウント**表示はオプション
- **+1で動画を自動一時停止**（オプション）
- 表示の位置/文字サイズ/全体倍率/内部余白/配色、加点ボタン色はチームカラーに連動
- **永続ストレージ要求＋前回プロジェクトの自動復元**（iOSの自動削除対策）
- プロジェクトの**複製**、動画再選択時の**ファイル名/長さ照合警告**
- **ASS字幕の書き出し**（装飾付き・ffmpegで直接焼き込み可）、JSON 保存/読込、書き出し情報出力

## ファイル構成
```
index.html              アプリ本体（4画面）
manifest.webmanifest    PWAマニフェスト
sw.js                   Service Worker（アプリシェルのみキャッシュ／動画は非キャッシュ）
css/styles.css          スタイル（縦画面優先・横画面対応）
js/
  app.js                ルーティング・結線・自動保存
  state.js              プロジェクトモデル・スコア算出
  events.js             加点/取消/セット/サーブ権ロジック
  storage.js            IndexedDB・JSON入出力
  video.js              動画選択・再生・シーク
  scoreboard.js         スコアボード重ね表示・ドラッグ
  export.js             書き出し用JSON・FFmpegヒント
icons/                  PWAアイコン（192/512）
```

## 使い方（操作フロー）
1. アプリを開く（PWAとしてホーム画面に追加可能）。
2. **新規プロジェクト** を作成。
3. **設定画面** でチーム名・色・スコアボードの位置/文字サイズ等を設定。
4. **編集画面** で「動画を選択」→ iPhone内の試合動画を選ぶ。
5. 動画を再生し、得点が入った瞬間に **自チーム +1 / 相手 +1** を押す。
6. 誤って押したら **取り消し**。セット終了で **次のセットへ**。
7. シークすると、その時刻のスコア表示がプレビューに追従します。
8. **履歴画面** で得点一覧を確認（時刻ジャンプ・削除）。
9. 編集内容は自動保存（IndexedDB）。**JSON 書き出し** で外部保存も可能。

## ローカルでの動作確認（PC）
PWA / ES Modules のため、`file://` ではなく **HTTP サーバ経由**で開く必要があります。

```bash
# このフォルダで
python -m http.server 8000
#   → http://localhost:8000/ を Chrome 等で開く

# もしくは
npx serve .
```

DevTools の Application タブで、Manifest 認識・Service Worker 登録・オフライン（Network を offline にして再読込）を確認できます。

## iPhone 実機での利用
PWA・カメラロール動画の読み込みには **HTTPS** が必要です（`localhost` を除く）。

1. このフォルダを静的ホスティングへデプロイ（例：GitHub Pages / Netlify / Cloudflare Pages）。
   - GitHub Pages の場合：リポジトリに push → Settings → Pages → ブランチを公開。
2. iPhone の Safari で公開 URL を開く。
3. 共有メニュー → **「ホーム画面に追加」**。
4. ホームのアイコンから起動（スタンドアロン表示）。
5. 「動画を選択」から写真ライブラリ／ファイルの動画を選んで編集。

> ローカルだけで実機確認したい場合は、同一 Wi-Fi 上の PC で HTTPS サーバ（自己署名証明書）を立て、`https://<PCのIP>:ポート` を iPhone Safari で開く方法もあります（証明書の信頼設定が必要）。

```jsonc
{
  "id": "…",
  "projectName": "2026-06-16 試合動画",
  "videoFileName": "match001.mp4",
  "videoDuration": 3600.5,
  "currentSet": 1,
  "teams": {
    "home": { "name": "自チーム", "color": "#1f6feb" },
    "away": { "name": "相手チーム", "color": "#f85149" }
  },

    "position": "top-left", "x": 16, "y": 16,

```

## PCでの動画への焼き込み（推奨：ASS字幕）
イベント一覧画面の **「ASS字幕を書き出し」** で、装飾付きのスコアボードを **ASS字幕ファイル（.ass）** として出力します。
チームカラー・改行・**全セットの併記（列）**・サーブ権の赤丸・表示位置/文字サイズ/背景を、プレビューに近い形で保持します。

ffmpeg でそのまま焼き込めます（**JSON→ASS等の変換は不要**）：
```bash
ffmpeg -i input.MOV -vf "ass=scoreboard.ass" -c:a copy output.mp4
```
- `.ass` と元動画を同じフォルダに置いて実行してください（パスを含める場合は ffmpeg のエスケープに注意）。
- フォントは既定で `Meiryo`（Windows標準）。別フォントにする場合は `.ass` の `Style:` 行の `Meiryo` を置き換えます。
- 位置・サイズは編集時のプレビュー枠を基準にしており、libass が動画解像度へ比例拡大します。動画を選択して長さ・解像度を取得済みのプロジェクトで書き出すと位置が正確になります。

> ASSはテキストベースのため、アプリ内プレビューの角丸枠などを**ピクセル単位で完全一致**はしません。完全一致が必要な場合は、得点状態ごとの透過PNGを `overlay` で重ねる方式（将来拡張）が候補です。

### その他の書き出し
- **JSON**：編集データそのもの（再読込・バックアップ用）。
- **書き出し情報**：時間区間付きセグメントと `drawtext` 参考ヒントを含む `*.export.json`（独自ツールでの加工用）。

## 動画の読み込みが遅いとき
本アプリは動画を**アップロードも複製もしません**（`URL.createObjectURL` でそのまま再生）。
それでも読み込みに時間がかかる場合、原因の多くは **iOS 側の処理**で、アプリの外で発生しています。

**主な原因**
- **写真ライブラリからの書き出し**：iPhoneのHEVC(H.265)/4K動画は、Safariに渡す際にiOSがH.264へ変換・書き出しすることがあり、30分動画では数十秒〜数分かかります。
- **iCloudからのダウンロード**：「設定 > 写真 > iPhoneのストレージを最適化」がオンだと、原本をiCloudから取得してから渡すため待たされます。

**速くするコツ**
1. **「ファイル」アプリのMP4を選ぶ**（写真ライブラリ経由を避ける）。すでにMP4化されていれば書き出しが起きず高速です。
2. **撮影フォーマットを互換性優先に**：設定 > カメラ > フォーマット >「互換性優先(H.264)」。変換が不要になります。
3. **事前にダウンロード**：対象動画を写真アプリで一度開き、iCloudから端末へ落としておく。

**切り分け方法（開発者向け）**
動画選択時、ブラウザのコンソールに次のログを出します。
- `選択→受領: Nms` … iOSの書き出し/iCloudダウンロード時間（ここが大きい＝iOS側がボトルネック）。
- `メタデータ解析: Nms` … Safariが長さ等を読む時間（ここが大きい＝動画形式/Safari側）。

読み込み中はスピナーと経過秒を表示するので、フリーズではないことを確認できます。

## 制限・対象外（初期版）
iPhone単体でのMP4書き出し／再エンコード／複数動画結合／音声編集／テロップアニメーション／クラウド同期 などは対象外です。
